# 🏦 Expense Tracker Bot - Telegram Bot для Учёта Расходов

Современный Telegram-бот для быстрого трекинга расходов с голосовым вводом, AI-категоризацией, проектами и интеграцией с Google Sheets.

## 🚀 Возможности

### 📱 Основные функции
- **🎤 Голосовой ввод** - Отправляйте голосовые сообщения для быстрого добавления расходов
- **🤖 AI-парсинг** - Автоматическое распознавание суммы, валюты и категории
- **📊 Умная категоризация** - AI определяет категории и запоминает ваши привычки
- **📋 Проекты** - Разделение расходов по проектам
- **📈 Google Sheets** - Автоматическая синхронизация с таблицами
- **🔍 AI-аналитика** - Задавайте вопросы о ваших тратах на естественном языке
- **💱 Мультивалютность** - Поддержка 8 основных валют без конвертации

### Возможности без подписки

- Несколько проектов и записей
- AI-вопросы и аналитика
- Синхронизация с Google Sheets
- Командная работа
- Кастомные категории

## 🛠 Технический стек

- **Backend**: Node.js + Express
- **Bot Framework**: node-telegram-bot-api
- **AI**: OpenAI API (Whisper для голоса + GPT-4 для парсинга и аналитики)
- **Database**: Supabase (PostgreSQL)
- **Sheets**: Google Sheets API
- **Storage**: Google Drive API

## 📦 Установка и настройка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd expense-tracker-bot
npm install
```

### 2. Настройка переменных окружения

Создайте `.env` файл на основе `.env.example`:

```bash
# Telegram Bot
BOT_TOKEN=your_telegram_bot_token

# OpenAI
OPENAI_API_KEY=your_openai_api_key

# Supabase
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_KEY=your_supabase_service_role_key

# Google Sheets
GOOGLE_CLIENT_EMAIL=your_service_account_email
GOOGLE_PRIVATE_KEY=your_service_account_private_key

# App Settings  
NODE_ENV=production
PORT=3000
WEBHOOK_URL=your_webhook_url
```

### 3. Настройка базы данных Supabase

1. Создайте проект в [Supabase](https://supabase.com)
2. Выполните SQL скрипт из `migrations/create_functions.sql`
3. Настройте Row Level Security политики

### 4. Настройка Google Sheets API

1. Создайте проект в [Google Cloud Console](https://console.cloud.google.com)
2. Включите Google Sheets API
3. Создайте Service Account и скачайте JSON ключ
4. Добавьте email Service Account в переменные окружения

### 5. Создание Telegram бота

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен и добавьте в `.env`
3. Настройте команды бота:

```
start - Начать работу с ботом
help - Справка по возможностям
```

### 6. Запуск

```bash
# Разработка
npm run dev

# Продакшн
npm start
```

## 🗄 Структура базы данных

### Основные таблицы
- `users` - Пользователи бота
- `projects` - Проекты пользователей
- `project_members` - Участники проектов (для командной работы)
- `expenses` - Записи о расходах
- `user_patterns` - Паттерны пользователя для Smart Defaults
- `custom_categories` - Кастомные категории

### Функции базы данных
- `get_monthly_stats()` - Статистика по месяцам
- `get_user_expenses_for_period()` - Расходы за период
- `increment_counter()` - Счётчики лимитов

## 🔒 Безопасность

- **Row Level Security (RLS)** - Пользователи видят только свои данные
- **Rate Limiting** - Лимиты на AI запросы и синхронизацию
- **Input Validation** - Валидация всех входящих данных
- **Error Handling** - Обработка всех ошибок без раскрытия системной информации

## 📊 Google Sheets интеграция

### Структура таблицы
- **Колонка A**: Дата (DD.MM.YYYY)
- **Колонка B**: Описание расхода
- **Колонка C**: Сумма
- **Колонка D**: Валюта
- **Колонка E**: Категория
- **Колонка F**: Автор (username)
- **Колонка G**: Источник (bot/manual)

### Синхронизация
- Автоматическое добавление новых расходов в Google Sheets
- Импорт записей, добавленных вручную в таблицу
- Проверка дубликатов по номеру строки
- Двусторонняя синхронизация с разрешением конфликтов

## 🤖 Smart Defaults (Умные предложения)

Система запоминает привычки пользователя:
- Анализирует ключевые слова в описаниях трат
- Запоминает частые категории для определенных покупок
- Предлагает суммы на основе истории
- Повышает точность со временем

**Алгоритм:**
1. Извлечение ключевых слов из описания
2. Поиск паттернов в истории пользователя
3. Расчёт уверенности на основе частоты и актуальности
4. Предложение категории, суммы и валюты
